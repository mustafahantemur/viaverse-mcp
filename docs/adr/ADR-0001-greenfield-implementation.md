# ADR-0001 — Greenfield Implementation

## Status

Accepted

## Context

Viaverse must be planned as a clean, production-shaped system. Prototype screenshots and earlier experiments are useful as product/visual references, but must not define the implementation architecture.

## Decision

Viaverse is a greenfield implementation.

Prototype screenshots are visual/product references only.

## Consequences

- No inherited package structure.
- No inherited database schema.
- No inherited API structure.
- No implementation assumption from any previous prototype.
- Product learnings can be reused, but code and architecture must be designed cleanly.
